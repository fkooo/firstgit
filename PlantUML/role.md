# Webアプリケーション開発プロジェクト（ウォーターフォール型） 役割分担表

本書は、当Webアプリケーション開発プロジェクトにおける各工程のタスクと、アクターごとの役割および責任範囲を定義したものです。
本版では、プロジェクト計画・実行の主体となる「プロマネ」「開発担当」およびリリース後の運用を担う「保守チーム」にフォーカスし、関わるタスクを細分化して記載しています。

## 1. アクター定義（本ドキュメントの対象）

| アクター | 概要・主な責任 |
| :--- | :--- |
| **プロマネ** | プロジェクトマネージャー（およびPMO）。プロジェクト全体の計画立案、進捗・課題・品質管理など、マネジメント全般に責任を持つ。 |
| **開発担当** | フロントエンド・バックエンド等アプリケーション領域の要件整理・設計・実装・テストの実務を行う。アプリの技術的品質を担保する。 |
| **保守チーム** | リリース後のシステム定常監視、サービスデスク、障害対応、運用改善などの保守業務を担当する。 |

（※お客様やインフラチーム等もプロジェクトには参画しますが、本表では主に上記3チームのアクションに特化しています）

## 2. 役割の定義（凡例）

| 記号 | 役割名称 | 定義 |
| :---: | :--- | :--- |
| **◎** | 主担当 (Responsible) | 該当タスクの実行主体であり、成果物の作成や推進に直接的な責任を持つ。 |
| **◯** | 副担当/支援 (Support) | 主担当を補佐し、実作業の一部を担う、もしくは進行の後方支援を行う。 |
| **△** | 承認/相談 (Approve/Consult)| 要件や成果物の確認・承認を行う。または専門的見地から意見・助言を提供する。 |
| **ー** | 関与なし (None) | 当該タスクには直接的な関与を持たない。 |

## 4. 特記事項・留意事項
* 本表は「プロマネ」「開発担当」「保守チーム」向けの実務フォーカス版です。お客様やインフラチームの「相手側」のタスク自体は省略していますが、連携・調整タスクは明記しています。
* 各テスト工程において、バグ（不具合）が発見された場合の起票やトリアージ方針は「プロマネ」が牽引し、調査・修正は「開発担当」が主体（◎）となって行います。
* 本番運用以降の不具合対応において、コード改修が必要な場合は「開発担当」が主体（◎）となりますが、障害受付および一次切り分けの主導は「保守チーム」が担います。
